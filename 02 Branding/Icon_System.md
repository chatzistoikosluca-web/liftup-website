# Icon-System — Schritt 5.8 (Strategie)

Status: **Final freigegeben (12.07.2026).** Basis: `Bildsprache.md` (Kategorie 5 —
Grafische Elemente), `Kreative_Markenrichtung.md` (Punkt 15), `LiftUp_Brand_Bible.md`
(UX-Prinzip 3), `Design_Verfassung.md` (Grundprinzip 2/5), `Design_Checkliste.md`
(Punkt 4). Bewusst minimal — Icons besitzen eine noch schwächere dokumentierte Grundlage
als Radius oder Schatten. Noch keine Stilentscheidung, keine konkreten Formen, keine
Icon-Sets, keine Größen.

## Strategischer Inhalt

**Existenzfrage bleibt offen.** Ob und wo Icons im LiftUp-System überhaupt gebraucht
werden, ist durch keines der Dokumente entschieden oder ausgeschlossen.

**Falls Icons verwendet werden, gelten ausschließlich bereits bestehende Regeln:**
1. Jedes Icon braucht eine benennbare Funktion (Orientierung, Zustand, Aktion) — reine
   Dekoration ist ausgeschlossen (Grundprinzip 2; Design-Checkliste Punkt 4).
2. Icons dürfen nicht als Ersatzbeweis für echte Substanz getarnt werden (Bildsprache,
   Kategorie 5) — z. B. kein Icon anstelle eines echten Zertifikats/Kundenlogos.
3. Sofern Icons verwendet werden, dürfen sie den bereits bestehenden Token-Systemen nicht
   widersprechen. Ob und in welchem Umfang Farbrollen oder Formprinzipien direkt
   übernommen werden, wird erst entschieden, wenn ein dokumentierter Bedarf für ein
   Icon-System besteht.
4. Icons dürfen keine der in Kreative Markenrichtung Punkt 15 bereits ausgeschlossenen
   Klischees zeigen — insbesondere Raketen-/Aufzug-/Pfeil-Motive, unabhängig vom
   Verwendungskontext.
5. Wo bereits eine andere Lösung dokumentiert ist (z. B. Nummerierung bei
   Prozessschritten, UX-Prinzip 3), ersetzt ein Icon diese nicht ohne gesonderten Nachweis
   eines Bedarfs (Grundprinzip 5).

**Nicht Gegenstand dieses Schritts (mangels jeder Grundlage):** Icon-Stil
(Linie/Fläche/Duotone), Strichstärke, eigenes vs. eingekauftes Icon-Set, konkrete Größen,
Übernahme von Farbrollen/Rundungslogik im Detail.
